# Import

The import pipeline ingests tenant catalogs into the PIM through three
complementary channels:

- **File upload** — drop a CSV or Excel file, parsed by the import worker.
- **Direct API** — POST a JSON array and get results back in the same request.
- **Queued API** — POST a JSON array and let a background worker apply it.

All three reuse the same `ImportSource` (field mappings, locked fields,
auto-publish rules) and produce an `ImportJob` you can poll through the
[Jobs API](/developers/pim/activity).

<Callout kind="info" title="Base path">
  All endpoints on this page are rooted at <code>/api/b2b/pim/import</code>.
</Callout>

## Authentication

Every endpoint below requires tenant authentication via
<code>requireTenantAuth</code>. Three credentials are accepted:

## Flow overview

1. **Create a source** once per feed via [POST /api/b2b/pim/sources](/developers/pim/sources).
   The source stores `field_mappings` (source → target) and auto-publish rules.
2. **Optionally analyze** a sample file with `POST /import/analyze` to inspect
   headers and infer types before committing to a mapping.
3. **Send data** through one of:
   - `POST /import` — multipart file upload, CDN-backed, background worker.
   - `POST /import/api` — synchronous JSON upsert, returns full results.
   - `POST /import/queue` — asynchronous JSON upsert, returns a `job_id`.
4. **Poll status** via [GET /api/b2b/pim/jobs/:jobId](/developers/pim/activity)
   for counters and per-row errors.

## Direct JSON import (synchronous)

<Endpoint method="POST" path="/api/b2b/pim/import/api" />

Applies an array of products inline in a single request and returns the
full result set. Ideal for ≤ 1 000 products per call; the hard limit is
**10 000**.

<Callout kind="warning" title="Requires 'import' permission">
  API keys must have the <code>import</code> permission. Session and Bearer
  auth work without additional scopes but still require a valid tenant.
</Callout>

### Request body

<ParamTable
  rows={[
    { name: "source_id", type: "string", required: true, description: "ImportSource whose field_mappings and auto-publish rules are applied." },
    { name: "products", type: "Product[]", required: true, description: "Array of product payloads. 1 ≤ length ≤ 10000." },
    { name: "merge_mode", type: "'replace' | 'partial'", description: "'replace' (default) overwrites the product version entirely; 'partial' deep-merges incoming fields onto the existing version.", default: "'replace'" },
    { name: "batch_id", type: "string", description: "Optional correlation id persisted on each product's source.batch_id." },
    { name: "batch_metadata", type: "object", description: "Optional { batch_id, batch_part, batch_total_parts, batch_total_items } — stored on the product for downstream aggregation." },
    { name: "channel_metadata", type: "object", description: "Accepted and echoed back in the debug block. Reserved for future routing; currently unused by the pipeline." },
  ]}
/>

### Per-product fields

Each entry in `products` is mapped through `source.field_mappings` first,
then the mapped object is persisted. `entity_code` (or `sku`) is
**required** and identifies the product across versions.

<ParamTable
  rows={[
    { name: "entity_code", type: "string", required: true, description: "Stable tenant-scoped id. Falls back to `sku` if omitted." },
    { name: "sku", type: "string", description: "Defaults to entity_code if omitted." },
    { name: "name", type: "string | MultiLangString", description: "Plain strings are auto-wrapped as { <defaultLang>: value }." },
    { name: "description", type: "string | MultiLangString", description: "Same auto-wrapping for multilingual fields." },
    { name: "price", type: "number", description: "Numeric price; per-channel overrides handled by the PIM." },
    { name: "images", type: "MediaRef[]", description: "Replaced wholesale when provided. Kept when omitted in partial mode." },
    { name: "media", type: "MediaRef[]", description: "Merged by type in partial mode: incoming types replace existing items of that type; other types are preserved." },
  ]}
/>

### Processing rules

- **Versioning** — every successful upsert creates a new `PIMProduct`
  version. The prior `isCurrent: true` row is demoted and its
  `locked_fields` are re-applied to the incoming payload.
- **Multilingual auto-wrap** — multilingual fields (e.g.{" "}
  <code>name</code>, <code>description</code>, <code>slug</code>) are
  wrapped as <code>{`{ <defaultLang>: value }`}</code> unless they already
  look like a language-keyed object (2-letter codes matching a language
  configured for the tenant).
- **Auto-publish** — a version with a sufficient completeness score is
  committed as `published` with `published_at` set. Otherwise it lands as
  `draft`.
- **Search sync** — when the tenant has a search-enabled default
  language, successful entity codes are batched (50 per batch) and
  queued for indexing into the tenant's search surface.

### Response

<ParamTable
  rows={[
    { name: "success", type: "boolean", description: "true even when some rows failed — check summary.failed." },
    { name: "job_id", type: "string", description: "ImportJob id, prefix 'api_import_'. Persisted for audit." },
    { name: "summary.total", type: "integer", description: "Rows attempted." },
    { name: "summary.successful", type: "integer", description: "Rows persisted." },
    { name: "summary.failed", type: "integer", description: "Rows rejected — see errors[]." },
    { name: "summary.auto_published", type: "integer", description: "Rows that met auto-publish criteria." },
    { name: "summary.duration_seconds", type: "number", description: "Server-side processing time." },
    { name: "summary.sync_batches_queued", type: "integer", description: "Number of search-sync batches queued (50 products per batch)." },
    { name: "errors[]", type: "{ row, entity_code, error, raw_data }", description: "First 100 failures, with the raw row that caused them." },
    { name: "debug", type: "object", description: "Echo of batch_id, batch_metadata, channel_metadata, and the first product's source block for inspection." },
  ]}
/>

## Queued JSON import (asynchronous)

<Endpoint method="POST" path="/api/b2b/pim/import/queue" />

Identical payload shape to `/import/api`, but the request returns
immediately after queueing. A background worker applies the same rules
asynchronously. Use this for large batches or when latency matters.

### Request body

<ParamTable
  rows={[
    { name: "source_id", type: "string", required: true, description: "ImportSource to apply." },
    { name: "products", type: "Product[]", required: true, description: "1 ≤ length ≤ 10000." },
    { name: "batch_metadata", type: "object", description: "Optional { batch_id, batch_part, batch_total_parts, batch_total_items }. Stored on the ImportJob and forwarded to the worker." },

/>

### Retry behavior

- Generated job id is returned in the response.
- Transient failures are retried automatically with exponential backoff.
- Persistent failures land on the job as `status: "failed"` with per-row
  errors in `import_errors`.

## Supported entities

The import worker is currently product-centric. Other entities are
imported through their dedicated endpoints:

- **products** — `/api/b2b/pim/import/*` (this page).
- **categories** — `/api/b2b/pim/categories/:id/import`.
- **brands** — `/api/b2b/pim/brands/:id/import`.
- **collections** — `/api/b2b/pim/collections/:id/import`.
- **product types** — `/api/b2b/pim/product-types/:id/import`.
- **synonym dictionaries** — `/api/b2b/pim/synonym-dictionaries/:id/import`.

See each entity's page for its specific payload.

## Idempotency and retries

Re-uploading the same file or re-sending the same array does **not**
deduplicate by itself. Rows match existing products by `entity_code`
(falling back to `sku`) and create a new version on every successful
apply.

<Callout kind="warning" title="Avoid double-applies">
  When retrying a failed request, pass the same <code>batch_id</code> —
  it lands on each product's <code>source.batch_id</code> and lets you
  reconcile with upstream audit logs.
</Callout>
