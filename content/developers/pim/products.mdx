operations, and export over a tenant's catalog.

</Callout>

## List products

Returns a paginated list of products, filtered and sorted by query params.

### Query parameters

  <code>entity_code</code> must be unique per tenant. Creating an existing
  one returns <code>409 Conflict</code>.
</Callout>

## Update a product

<Endpoint method="PATCH" path="/api/b2b/pim/products/:entity_code" />

Partial update — only the fields you send are touched. The server merges
deeply for nested objects (e.g. `attributes`, `channel_overrides`) and
replaces arrays whole.

### Updatable field groups

- **Basic:** `name`, `description`, `status`, `brand`, `categories`, `tags`.
- **Associations:** `parent_sku`, `variants`, `collections`.
- **Attributes:** typed `attributes.<code>` values driven by the product type.
- **Pricing:** `price`, `tiered_prices`, `cost`, `msrp`.
- **Media:** `images`, `videos`, `documents` (as ordered lists of CDN refs).
- **Channels:** per-channel overrides for marketplace and storefront.

## Publish / unpublish

<Endpoint method="POST" path="/api/b2b/pim/products/:entity_code/publish" />
<Endpoint method="POST" path="/api/b2b/pim/products/:entity_code/unpublish" />

Publishing flips `status` to `published` and enqueues a search-index
update job. Unpublishing reverses both.

## Bulk operations

<Endpoint method="POST" path="/api/b2b/pim/products/bulk-publish" />
<Endpoint method="POST" path="/api/b2b/pim/products/bulk-update" />
<Endpoint method="POST" path="/api/b2b/pim/products/export" />

Bulk endpoints accept either an explicit array of `entity_code`s or a filter
matching the list-endpoint query params. Every bulk call returns a
`job_id` you can poll via [Activity](/developers/pim/activity).

<CodeBlock
  title="POST /api/b2b/pim/products/bulk-publish"
  code={`{
  "filter": { "status": "draft", "brand": "acme" }
}

// Response
{
  "success": true,
  "job_id": "pim-bulk-publish-01H7X..."
}`}
/>
