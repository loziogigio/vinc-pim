# Companies

Push your customer base into a tenant in bulk. The endpoint accepts an array
of **company** records — each with its legal data and an embedded list of
addresses — validates the payload, enqueues a background job, and returns a
`job_id` you can poll. Records are matched on `external_code`, so re-sending
the same feed updates the existing companies (idempotent upsert).

<Callout kind="info" title="Base path">
  The endpoints on this page are rooted at <code>/api/b2b/customers/import</code>.
</Callout>

<Callout kind="tip" title="Import order">
  Import companies **before** portal users. A portal user links to a company
  by its `external_code`, which is resolved server-side — see
  [Portal Users](/developers/customers/portal-users).
</Callout>

## Authentication

Both endpoints accept either a tenant API key or a B2B portal session cookie.

## Import companies

<Endpoint method="POST" path="/api/b2b/customers/import/api" auth="api-key" />

Queues an array of companies for upsert. Returns `202 Accepted` immediately;
the actual write happens in a background worker.

### Constraints

- 1 ≤ `customers.length` ≤ **5000** per request.
- Every entry must carry a non-empty `external_code`.
- Recommended batch size: **≤ 500** companies per call; split larger feeds
  into multiple requests sharing a `batch_metadata.batch_id`.

### Request body

<ParamTable
  rows={[
    { name: "customers", type: "CustomerImportItem[]", required: true, description: "Array of company payloads — see the field table below." },
    { name: "merge_mode", type: "'replace' | 'partial'", description: "'replace' overwrites every provided field (and the whole address list when addresses is sent); 'partial' deep-merges only the fields you send and matches addresses by external_code.", default: "'replace'" },
    { name: "channel", type: "string", description: "Default sales channel for companies created in this batch. A per-company channel takes priority. Falls back to the tenant default channel." },
    { name: "batch_metadata", type: "object", description: "Optional { batch_id, batch_part, batch_total_parts, batch_total_items } — persisted on the ImportJob so multi-part feeds can be correlated." },
  ]}
/>

<ParamTable

    { name: "company_name", type: "string", description: "Legal / trade name (ragione sociale)." },
    { name: "first_name", type: "string", description: "Contact first name — useful for private customers." },
    { name: "last_name", type: "string", description: "Contact last name." },
    { name: "email", type: "string", description: "Primary email for the company." },
    { name: "phone", type: "string", description: "Primary phone." },
    { name: "legal_info", type: "object", description: "{ vat_number, fiscal_code, pec_email, sdi_code } — all optional, each validated when present (see below)." },
    { name: "tags", type: "string[]", description: "Free-form labels. In partial mode tags are added (additive upsert); in replace mode they overwrite. Used for filtering, segmentation, and price-list rules." },
    { name: "addresses", type: "CustomerImportAddress[]", description: "Embedded delivery / billing addresses — see the address field table." },
  ]}
/>

<Callout kind="info" title="legal_info validation">
  When a field is present it must be well-formed, otherwise the company row is
  rejected: <code>vat_number</code> → <code>IT</code> + 11 digits;{" "}
  <code>fiscal_code</code> → 16 chars (individuals) or 11 digits (companies);{" "}
  <code>sdi_code</code> → 7 alphanumeric chars; <code>pec_email</code> → a
  valid email. Omit a field rather than send an empty / unknown value.
</Callout>

<Callout kind="warning" title="Invalid addresses are skipped, not fatal">
  An address missing <code>street_address</code>, <code>city</code>, or{" "}
  <code>province</code> is dropped (a warning is logged) and the company is
  still imported. In <code>partial</code> mode an address with an{" "}
  <code>external_code</code> that matches an existing one is merged field-by-field
  (a label-only update is fine); an unmatched <code>external_code</code> creates
  a new address if its required fields are present.
</Callout>

### Response — `202 Accepted`

<ParamTable
  rows={[
    { name: "success", type: "boolean", description: "true when the batch was accepted and queued." },
    { name: "job_id", type: "string", description: "ImportJob id, prefix 'cust_import_'. Poll it for progress and per-row errors." },
    { name: "total", type: "integer", description: "Number of companies queued." },
    { name: "merge_mode", type: "'replace' | 'partial'", description: "Echo of the mode applied." },
    { name: "message", type: "string", description: "Human-readable confirmation." },
  ]}
/>

`400` is returned for an empty / oversized `customers` array, an invalid
`merge_mode`, or a company missing `external_code` (the error names the offending
index). Per-row failures during processing (e.g. bad `legal_info`) do **not**
fail the request — they surface on the job under `import_errors`.

### partial vs replace

<CodeBlock
  title="merge_mode: 'partial' — preserves omitted fields & merges addresses by external_code"
  code={`// Existing company:
// { external_code: "000123", company_name: "ACME S.R.L.", email: "old@acme.it", addresses: [{ external_code: "000000", city: "Napoli", ... }] }

// Incoming:
{ "external_code": "000123", "email": "info@acme.it",
  "addresses": [{ "external_code": "000000", "label": "HQ - Napoli (NA)" }] }

// Result: email updated, address 000000 keeps its fields with the new label, every other field untouched.`}
/>

<CodeBlock
  title="merge_mode: 'replace' (default) — provided fields win, addresses replaced wholesale"
  code={`// Same existing company, same incoming payload:
// email is updated; the addresses array is replaced by [{ external_code: "000000", label: "HQ - Napoli (NA)" }]
// — and that address is dropped because it is missing street_address / city / province.`}
/>

## Check job status

<Endpoint method="GET" path="/api/b2b/customers/import/api/{job_id}" auth="api-key" />

Returns the `ImportJob` created by the POST above. Poll until `status` is
`completed` or `failed`.

<ParamTable
  rows={[
    { name: "job.job_id", type: "string", description: "Echoes the requested id." },

    { name: "job.total_rows", type: "integer", description: "Companies in the batch." },
